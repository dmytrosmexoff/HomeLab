# SA-MP Server Monitor — установка

Полноценное приложение для Umbrel: хостинг SA-MP сервера + веб-панель управления
с реальными данными (без случайных/тестовых значений).

⚠️ **Важно:** `samp03svr` — 32-битный x86-бинарник. Приложение соберётся и будет
работать только на **amd64/x86_64** хостах Umbrel. На ARM-устройствах (Raspberry Pi
и т.п.) SA-MP сервер запустить не получится.

## 1. Репозиторий с исходным кодом (сборка образов)

1. Создай новый **публичный** репозиторий на GitHub, например `samp-monitor`.
2. Залей туда содержимое папок `api/`, `web/`, `.github/` из этого архива (без папки `homelab-app-store`).
3. Открой `.github/workflows/build.yml` — он уже настроен, ничего менять не нужно:
   при любом пуше в `main` GitHub Actions сам соберёт оба образа
   (`samp-monitor-api`, `samp-monitor-web`) и запушит их в
   `ghcr.io/<твой_github_ник>/...`.
4. Убедись, что в настройках репозитория включены **GitHub Actions**
   (Settings → Actions → General → разрешить запуск workflow).
5. Первый пуш в `main` запустит сборку — проверить статус можно во вкладке **Actions**.
6. После первой успешной сборки зайди в **Packages** твоего GitHub-аккаунта и
   у обоих пакетов (`samp-monitor-api`, `samp-monitor-web`) поставь
   видимость **Public** (иначе Umbrel не сможет их скачать анонимно).

## 2. Добавление в твой App Store `homelab`

1. В существующем репозитории `homelab` (id: `homelab`, name: "Home Lab")
   скопируй папку `homelab-app-store/samp-monitor` из этого архива в корень репозитория,
   рядом с остальными приложениями — то есть должно получиться `homelab/samp-monitor/`.
2. В файлах `homelab/samp-monitor/umbrel-app.yml` и `docker-compose.yml`
   замени все `ТВОЙ_GITHUB_НИК` на свой реальный GitHub-логин
   (тот же, под которым собрались образы в шаге 1).
3. Закоммить и запушь изменения в `homelab`.

## 3. Установка на Umbrel

1. На Umbrel: **Настройки → App Store → Community App Stores** → добавить URL
   твоего репозитория `homelab` (если он ещё не добавлен).
2. Найди "SA-MP Server Monitor" в разделе Home Lab и нажми **Install**.
   Umbrel сам скачает оба образа с `ghcr.io` — никаких ручных действий не требуется.
3. Если нужно зайти по SSH и проверить/перезапустить приложение вручную:
   ```
   sudo ~/umbrel/scripts/app compose samp-monitor logs -f
   sudo ~/umbrel/scripts/app compose samp-monitor restart
   ```
4. Данные сервера (gamemode, filterscripts, логи, аккаунты) хранятся в
   ```
   sudo ls ~/umbrel/app-data/samp-monitor/data
   ```
   и переживают перезапуск/обновление контейнера.

## 4. Первый запуск приложения

1. Открой панель на `http://<IP_твоего_Umbrel>:2323`.
2. На экране настройки загрузи `.amx` файл своего gamemode (и, при желании,
   filterscripts), укажи имя сервера, RCON-пароль и макс. игроков → «Сохранить».
3. Нажми **Старт** — сервер реально запустится внутри контейнера,
   в консоли появится настоящий вывод процесса, а карточки (игроки, CPU, RAM,
   аптайм) начнут показывать реальные данные, полученные по SA-MP Query Protocol
   и из `/proc` запущенного процесса.
4. Игроки подключаются к твоему серверу по адресу `<IP_твоего_Umbrel>:7777`.

## Обновление приложения в будущем

Просто пуш изменений в `main` репозитория `samp-monitor` → Actions пересоберёт
образы с тегом `:latest` → на Umbrel нажимаешь **Update** у приложения.
